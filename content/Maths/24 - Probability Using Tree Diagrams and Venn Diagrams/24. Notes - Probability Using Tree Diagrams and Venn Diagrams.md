## Tree Diagrams

A **tree diagram** maps out all possible outcomes of two or more successive events.

### Structure
- Each **branch** represents one outcome
- The **probability** is written on each branch
- Probabilities on branches from the same point **add to 1**
- To find the probability of a combined outcome: **multiply along branches**
- To find the probability of multiple outcomes: **add the products**

> [!example] Bag A: 3 red, 2 blue. Draw one ball, replace, draw again.
>
> P(R then R) = 3/5 × 3/5 = 9/25
> P(R then B) = 3/5 × 2/5 = 6/25
> P(B then R) = 2/5 × 3/5 = 6/25
> P(B then B) = 2/5 × 2/5 = 4/25
> Total = 25/25 = 1 ✓

> [!tip] Always check: the sum of all end probabilities = 1.

---

## With Replacement vs Without Replacement

### With Replacement
After each draw, the item is put back. **Probabilities stay the same** on each branch.

> [!example] Bag: 4 red, 6 blue. P(same colour twice, with replacement):
> P(RR) = 4/10 × 4/10 = 16/100
> P(BB) = 6/10 × 6/10 = 36/100
> P(same) = 52/100 = **13/25**

### Without Replacement
The item is NOT put back. **Probabilities change** on the second branch (denominators decrease).

> [!example] Bag: 4 red, 6 blue. P(2 red, without replacement):
> P(R₁) = 4/10
> P(R₂ | R₁) = 3/9 (one fewer red, one fewer total)
> P(RR) = 4/10 × 3/9 = 12/90 = **2/15**

> [!warning] In "without replacement" problems — the denominator decreases by 1, and the numerator also decreases by 1 only if the same type was drawn.

## Conditional Probability

$$P(A | B) = \frac{P(A \cap B)}{P(B)}$$

Read as "probability of A **given** that B has already occurred."

> [!example] P(A ∩ B) = 0.12, P(B) = 0.4. Find P(A|B).
> P(A|B) = 0.12/0.4 = **0.3**

> [!example] From a tree diagram: given a red was drawn first, find P(red second).
> This is asking for P(R₂ | R₁) — just read from the conditional branch.

## Venn Diagrams for Probability

### Two Sets

The universal set ξ has n(ξ) elements.

| Region | Meaning | Count |
|--------|---------|-------|
| Only A | In A but not B | n(A) − n(A∩B) |
| Only B | In B but not A | n(B) − n(A∩B) |
| A ∩ B | In both | n(A∩B) |
| Neither | Outside both circles | n(ξ) − n(A∪B) |

### Key Formulas (from Venn Diagrams)
$$P(A \cup B) = P(A) + P(B) - P(A \cap B)$$

$$P(A | B) = \frac{P(A \cap B)}{P(B)}$$

> [!example] ξ = 40 students. A = likes football (25). B = likes cricket (18). A∩B = 8.
> n(A only) = 17, n(B only) = 10, n(both) = 8, n(neither) = 5
> P(A ∪ B) = 35/40 = **7/8**
> P(A | B) = (8/40)/(18/40) = 8/18 = **4/9**

### Three-Set Venn Diagrams
Fill in from the **centre outwards**:
1. Start with the triple intersection (A ∩ B ∩ C)
2. Fill in the pairwise intersections, subtracting the triple
3. Fill in the solo regions, subtracting all overlaps
4. Check total = n(ξ)

> [!example] n(A∩B∩C) = 3, n(A∩B) = 8, n(B∩C) = 6, n(A∩C) = 5, n(A) = 20, n(B) = 22, n(C) = 18, n(ξ) = 50
> A only = 20 − (5 + 8 − 3) − 3 = 20 − 10 − 3 = 7
> (Continue similarly for B and C, then check totals)

## Mutually Exclusive Events (Recap)

If A and B are mutually exclusive: P(A ∩ B) = 0

→ P(A ∪ B) = P(A) + P(B)

---

## Independent Events (Recap)

If A and B are independent: P(A ∩ B) = P(A) × P(B)

→ Also: P(A | B) = P(A) (knowing B happened doesn't change P(A))

---

## Combined Strategy

| Situation | Best method |
|-----------|------------|
| Two or more sequential events | Tree diagram |
| Events described by sets / groups | Venn diagram |
| "Given that…" probability | Conditional probability formula |
| Equally likely outcomes | Sample space / table |

---

## Key Definitions

| Term | Definition |
|------|-----------|
| Tree diagram | Diagram showing all outcomes of successive events with branch probabilities |
| With replacement | Item returned before next draw; probabilities unchanged |
| Without replacement | Item not returned; probabilities change each draw |
| Conditional probability | P(A|B) = probability of A given B has occurred |
| Independent | Knowledge of one event doesn't affect probability of the other |
| Mutually exclusive | Events that cannot happen at the same time; P(A∩B) = 0 |
| Venn diagram | Overlapping circles in a rectangle representing sets and their relationships |
