## 3.1 The Role of the CPU

> The **CPU** (Central Processing Unit) is the brain of the computer — it carries out all instructions from programs.

The CPU contains:
- **Control Unit (CU)** — directs the flow of data; controls the fetch-decode-execute cycle
- **Arithmetic and Logic Unit (ALU)** — performs arithmetic (+ - × ÷) and logical operations (AND, OR, NOT, comparisons)
- **Registers** — tiny, very fast memory stores inside the CPU
- **Cache** — fast memory between CPU and RAM

### Key Registers

| Register | Full name | Purpose |
|----------|-----------|---------|
| **MAR** | Memory Address Register | Holds the memory address currently being read/written |
| **MDR** | Memory Data Register | Holds data just read from or about to be written to memory |
| **PC** | Program Counter | Holds address of the **next** instruction to be fetched |
| **ACC** | Accumulator | Holds the result of the current calculation in the ALU |
| **CIR** | Current Instruction Register | Holds the instruction currently being decoded and executed |

---

## 3.2–3.4 The Fetch-Decode-Execute Cycle

Every instruction is processed in three stages:

### 1. Fetch
1. Address in PC copied to MAR
2. Data at that address fetched from RAM into MDR
3. Instruction in MDR copied to CIR
4. PC incremented (ready for next instruction)

### 2. Decode
- CU decodes the instruction in CIR
- Identifies the **opcode** (operation) and **operand** (data/address)

### 3. Execute
- CU or ALU carries out the instruction
- Result stored in ACC or written back to memory

## 3.5 Factors Affecting CPU Performance

| Factor | How it affects performance |
|--------|--------------------------|
| **Clock speed** (GHz) | More cycles per second → faster processing. 1 GHz = 1 billion cycles/sec |
| **Number of cores** | Multiple cores can execute different instructions simultaneously (parallel processing) |
| **Cache size** | Larger cache → more frequently used data available without going to slower RAM |
| **RAM amount** | More RAM → can hold more data for CPU to access quickly; less swapping to disk |

> **Overclocking:** Running a CPU at a higher clock speed than intended — increases performance but generates more heat and risks instability.

## 3.6 Input and Output Devices

### Input Devices

| Device | Data type | How it works |
|--------|-----------|-------------|
| **Keyboard** | Text | Keys pressed generate electrical signals mapped to character codes |
| **Mouse** | Position/clicks | Optical sensor detects movement; buttons send signals |
| **Microphone** | Sound | Converts sound waves to electrical signals |
| **Scanner** | Images | Light reflected off document → sensors → bitmap |
| **Digital camera** | Images | Light falls on CCD/CMOS sensor → pixel data |
| **Touchscreen** | Touch position | Capacitive/resistive — detects touch coordinates |
| **Barcode reader** | Barcode data | Laser reflects off bars/spaces → binary pattern |
| **QR code reader** | 2D code | Camera captures pattern → decodes to data |
| **2D/3D scanner** | 3D shape data | Laser measures distances to create 3D model |

### Output Devices

| Device | Output type | How it works |
|--------|------------|-------------|
| **Monitor (LCD/OLED)** | Images/video | Pixels lit up at varying colours |
| **Printer (inkjet)** | Paper documents | Sprays ink droplets; colour mixing via CMYK |
| **Printer (laser)** | Paper documents | Laser charges drum → toner applied → heat fuses |
| **3D printer** | Physical objects | Builds up layers of material (filament/resin) |
| **Speakers** | Sound | Converts digital signal → analogue → sound waves |
| **Actuator** | Physical movement | Motor/servo controlled by computer signals |

## 3.7 Data Storage

### Types of Storage

| Type | Full name | Examples | Speed | Volatile? |
|------|-----------|---------|-------|---------|
| **RAM** | Random Access Memory | DDR4 | Very fast | Yes |
| **ROM** | Read-Only Memory | BIOS chip | Fast | No |
| **HDD** | Hard Disk Drive | Spinning magnetic disks | Moderate | No |
| **SSD** | Solid-State Drive | Flash memory chips | Very fast | No |
| **Optical** | CD/DVD/Blu-ray | — | Slow | No |
| **Flash** | USB drives, SD cards | Memory cards | Fast | No |

### RAM vs ROM

| | RAM | ROM |
|--|-----|-----|
| **Volatile?** | Yes (lost when power off) | No (permanent) |
| **Writable?** | Yes (read and write) | No (read only) |
| **Used for** | Currently running programs and data | Firmware/BIOS |

### Secondary Storage — HDD vs SSD

| | HDD | SSD |
|--|-----|-----|
| Speed | Slower | Much faster |
| Moving parts | Yes (spinning disk + read head) | No |
| Durability | Less durable | More durable |
| Noise | Makes noise | Silent |
| Cost per GB | Cheaper | More expensive |
| Fragmentation | Can fragment | Not affected |

## 3.8 Virtual Memory

> **Virtual memory** is a section of secondary storage (hard disk) used as additional RAM when physical RAM is full.

- OS moves least-used data from RAM to a **paging/swap file** on disk
- Allows more programs to run than RAM alone could support
- Disadvantage: much slower than RAM → **thrashing** if too much virtual memory is used
- **Thrashing:** constant paging in/out causes major slowdown

---

## 3.9 Cloud Storage

> **Cloud storage** stores data on remote servers accessed via the internet, rather than on local devices.

### Advantages
- Accessible from any device, anywhere
- Data backed up automatically
- No physical storage device needed
- Easy to share files

### Disadvantages
- Requires internet connection
- Privacy/security risks (data held by third party)
- Ongoing subscription costs
- Slower than local storage

## 3.10 Network Hardware

| Device | Purpose |
|--------|---------|
| **Router** | Routes packets between different networks; connects home/office to the internet |
| **Switch** | Connects devices within a LAN; sends data only to the intended recipient |
| **Hub** | Like a switch but broadcasts to all devices — older, less efficient |
| **Network Interface Card (NIC)** | Hardware in a device that allows it to connect to a network |
| **Wireless Access Point (WAP)** | Connects wireless devices to a wired network |
| **Firewall** | Filters incoming/outgoing traffic based on rules; blocks unauthorised access |
| **Modem** | Modulates/demodulates signals to convert between digital and analogue for internet |

### Transmission Media

| Type | Speed | Distance | Use |
|------|-------|----------|-----|
| **Copper (Ethernet)** | Up to 10 Gbps | Limited by interference | LANs, short distances |
| **Fibre optic** | Up to 100+ Gbps | Very long distances | Backbone networks, long-haul |
| **Wi-Fi (wireless)** | Up to several Gbps | Limited by walls/interference | Mobile devices, homes |
