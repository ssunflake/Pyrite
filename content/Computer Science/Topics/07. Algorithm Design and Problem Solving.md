## 7.1 Program Development Life Cycle

> A structured approach to developing software through defined stages.

```
Analysis → Design → Coding → Testing
           ↑________________________|  (iterative)
```

| Stage | What happens |
|-------|-------------|
| **Analysis** | Understand and define the problem; identify inputs, processes, outputs |
| **Design** | Plan the solution using structure diagrams, flowcharts, pseudocode |
| **Coding** | Write the program in a chosen language |
| **Testing** | Test with varied data; find and fix errors; repeat if needed |

---

## 7.2 Analysis

- **Decompose** the problem: break it into smaller sub-problems
- Identify **inputs**, **processes**, and **outputs** (IPO)

**Example — Login System:**
- Inputs: username, password
- Processes: check username exists, verify password matches
- Outputs: "Login successful" or "Invalid credentials"

---

## 7.3 Design

### Decomposition and Structure Diagrams

> **Decomposition** is breaking a complex problem into smaller, manageable sub-problems that can each be solved independently.

A **structure diagram** shows the hierarchy of sub-problems.

```
          Login System
         /           \
    Get Input     Verify Input
   /        \     /          \
 Username  Password  Check User  Check Pass
```

### Flowcharts

Standard symbols:

| Symbol | Shape | Meaning |
|--------|-------|---------|
| **Start / Stop** | Rounded rectangle (oval) | Beginning or end |
| **Process** | Rectangle | An action or calculation |
| **Input / Output** | Parallelogram | Reading input or displaying output |
| **Decision** | Diamond | Yes/No question (branches) |
| **Arrow** | → | Direction of flow |

**Example flowchart — Check if a number is > 10:**

```
START
  ↓
[INPUT number]
  ↓
< number > 10? > -- NO --> [OUTPUT "Not more than 10"]
  | YES                              ↓
  ↓                                STOP
[OUTPUT "Greater than 10"]
  ↓
STOP
```

## 7.4 Pseudocode

> **Pseudocode** is code-like statements that describe an algorithm clearly but are not in any specific programming language.

Pseudocode uses familiar programming keywords (INPUT, OUTPUT, IF, FOR, WHILE) but does not have strict syntax rules.

**Example: Check if number < 10**

```
INPUT Number1
IF Number1 < 10 THEN
    OUTPUT "too small"
ELSE
    OUTPUT "valid"
ENDIF
```

The same logic in Python, VB.NET and Java:

```python
# Python
number1 = int(input())
if number1 < 10:
    print("too small")
else:
    print("valid")
```

```vb
' VB.NET
Dim number1 As Integer
number1 = Console.ReadLine()
If number1 < 10 Then
    Console.WriteLine("too small")
Else
    Console.WriteLine("valid")
End If
```

```java
// Java
Scanner scanner = new Scanner(System.in);
int number1 = Integer.parseInt(scanner.nextLine());
if (number1 < 10) {
    System.out.println("too small");
} else {
    System.out.println("valid");
}
```

## 7.6 Testing

> Testing verifies a program is correct, doesn't crash, and meets all requirements.

### Types of Test Data

| Type | Description | Example (age 10–100) |
|------|-------------|----------------------|
| **Normal** | Valid data the program should accept | 25, 50, 75 |
| **Abnormal** | Invalid data the program should reject | -1, 200, "abc" |
| **Extreme** | Edge of the valid range | 10, 100 |
| **Boundary** | Just inside and just outside the valid range | 9, 10, 100, 101 |

## 7.7 Common Algorithms

### Totalling

Initialise total to 0, add each value:

```python
# Python
total = 0
for i in range(5):
    value = int(input("Enter value: "))
    total = total + value
print("Total:", total)
```

```vb
' VB.NET
Dim total As Integer = 0
For i = 1 To 5
    Dim value As Integer = Console.ReadLine()
    total = total + value
Next
Console.WriteLine("Total: " & total)
```

```java
// Java
int total = 0;
Scanner sc = new Scanner(System.in);
for (int i = 0; i < 5; i++) {
    int value = Integer.parseInt(sc.nextLine());
    total = total + value;
}
System.out.println("Total: " + total);
```

### Finding Maximum / Minimum

- For **maximum**: initialise with a very small number (e.g. -1 or -999)
- For **minimum**: initialise with a very large number (e.g. 999)

```python
# Python - find maximum
largest = -1
for i in range(5):
    value = int(input())
    if value > largest:
        largest = value
print("Largest:", largest)
```

```vb
' VB.NET
Dim largest As Integer = -1
For i = 1 To 5
    Dim value As Integer = Console.ReadLine()
    If value > largest Then
        largest = value
    End If
Next
Console.WriteLine("Largest: " & largest)
```

```java
// Java
int largest = -1;
Scanner sc = new Scanner(System.in);
for (int i = 0; i < 5; i++) {
    int value = Integer.parseInt(sc.nextLine());
    if (value > largest) { largest = value; }
}
System.out.println("Largest: " + largest);
```

### Linear Search

Check each item one by one until found (or end of list):

```python
# Python
data = [5, 3, 9, 4, 7, 2]
searchValue = int(input("Search for: "))
found = False
for i in range(len(data)):
    if data[i] == searchValue:
        print("Found at index", i)
        found = True
        break
if not found:
    print("Not found")
```

```vb
' VB.NET
Dim data() As Integer = {5, 3, 9, 4, 7, 2}
Dim searchValue As Integer = Console.ReadLine()
Dim found As Boolean = False
For i = 0 To data.Length - 1
    If data(i) = searchValue Then
        Console.WriteLine("Found at index " & i)
        found = True
        Exit For
    End If
Next
If Not found Then Console.WriteLine("Not found")
```

```java
// Java
int[] data = {5, 3, 9, 4, 7, 2};
Scanner sc = new Scanner(System.in);
int searchValue = Integer.parseInt(sc.nextLine());
boolean found = false;
for (int i = 0; i < data.length; i++) {
    if (data[i] == searchValue) {
        System.out.println("Found at index " + i);
        found = true;
        break;
    }
}
if (!found) System.out.println("Not found");
```

### Bubble Sort

Compare adjacent pairs; swap if wrong order. Repeat until no swaps made.

```python
# Python (ascending order)
data = [6, 2, 5, 1]
n = len(data)
for passes in range(n - 1):
    changed = False
    for i in range(n - 1):
        if data[i] > data[i + 1]:
            # swap
            temp = data[i]
            data[i] = data[i + 1]
            data[i + 1] = temp
            changed = True
    if not changed:
        break  # already sorted
print(data)
```

```vb
' VB.NET
Dim data() As Integer = {6, 2, 5, 1}
Dim n As Integer = data.Length
For passes = 0 To n - 2
    Dim changed As Boolean = False
    For i = 0 To n - 2
        If data(i) > data(i + 1) Then
            Dim temp As Integer = data(i)
            data(i) = data(i + 1)
            data(i + 1) = temp
            changed = True
        End If
    Next
    If Not changed Then Exit For
Next
```

```java
// Java
int[] data = {6, 2, 5, 1};
int n = data.length;
for (int passes = 0; passes < n - 1; passes++) {
    boolean changed = false;
    for (int i = 0; i < n - 1; i++) {
        if (data[i] > data[i + 1]) {
            int temp = data[i];
            data[i] = data[i + 1];
            data[i + 1] = temp;
            changed = true;
        }
    }
    if (!changed) break;
}
```

**Bubble sort trace (ascending): 6, 2, 5, 1**

| Pass | Data after pass | Changes? |
|------|----------------|---------|
| Pass 1 | 2, 5, 1, 6 | Yes |
| Pass 2 | 2, 1, 5, 6 | Yes |
| Pass 3 | 1, 2, 5, 6 | Yes |
| Pass 4 | 1, 2, 5, 6 | No → Stop |

## 7.8 Trace Tables

> A **trace table** is used to dry-run an algorithm — follow each step manually to check the values of variables and outputs.

**Columns:** One for each variable + one for output
**Rows:** One for each step where a variable changes

**Example: trace with input 1, 2**

```
01 INPUT Number1
02 INPUT Number2
03 IF Number1 > Number2 THEN
04     OUTPUT Number1
05 ELSE
06     OUTPUT Number2
07 ENDIF
```

| Number1 | Number2 | OUTPUT |
|---------|---------|--------|
| 1 | — | — |
| 1 | 2 | — |
| — | — | 2 |

## 7.9–7.10 Finding Purpose and Errors in Algorithms

### Finding the Purpose
1. Test with sample data using a trace table
2. Observe what values change and what is output
3. Identify the pattern → state the purpose

### Finding Errors
Common error types:

| Type | Description | Example |
|------|-------------|---------|
| **Syntax error** | Code doesn't follow language rules | Missing colon in Python |
| **Logic error** | Code runs but gives wrong result | Using `>` instead of `<` |
| **Runtime error** | Crashes during execution | Dividing by zero |

**Approach to finding errors:**
1. Know what the program *should* do
2. Trace with test data → find where the output diverges from expected
3. Identify and fix the incorrect line

## Key Definitions

| Term | Definition |
|------|-----------|
| **Algorithm** | A set of step-by-step instructions to solve a problem |
| **Decomposition** | Breaking a complex problem into smaller sub-problems |
| **Pseudocode** | Code-like statements describing an algorithm without strict syntax |
| **Flowchart** | Diagrammatic representation of an algorithm using standard symbols |
| **Trace table** | Manual step-by-step tracking of variable values through an algorithm |
| **Linear search** | Checks each item in turn until found or end reached |
| **Bubble sort** | Repeatedly compares adjacent pairs and swaps if in wrong order |
| **Test data** | Input used to test a program; includes normal, abnormal, extreme, boundary |
