## 4.1 Types of Software

> **Software** = programs that tell the hardware what to do. Two main categories:

| Category | Description | Examples |
|----------|------------|---------|
| **System software** | Manages the hardware and provides a platform for applications | OS, utility software, drivers |
| **Application software** | Programs that perform tasks for the user | Word processors, browsers, games |

### Application Software Types

| Type | Purpose | Examples |
|------|---------|---------|
| **General purpose** | Wide range of tasks | Word processing, spreadsheets, browsers |
| **Special purpose** | One specific task | Barcode scanner software, medical imaging |
| **Bespoke** | Custom-built for one organisation | Bank's internal system, hospital patient DB |

### Off-the-shelf vs Bespoke Software

| | Off-the-shelf | Bespoke |
|--|--------------|---------|
| **Cost** | Cheaper (shared development cost) | Expensive (built from scratch) |
| **Time** | Available immediately | Takes time to develop |
| **Features** | General — may have unwanted features or lack specific ones | Exactly what is needed |
| **Support** | Ongoing vendor support | Support from developer |
| **Security** | Widely known vulnerabilities | Less widely attacked |

## 4.2 Operating System

> The **Operating System (OS)** is system software that manages the computer's hardware and provides services to application software. It acts as an **interface** between user/programs and hardware.

### Main Functions of an OS

| Function | Description |
|----------|-------------|
| **Process management** | Controls which programs run and shares CPU time between them |
| **Memory management** | Allocates RAM to programs; manages virtual memory |
| **File management** | Creates, stores, organises, retrieves and deletes files |
| **Device management** | Manages communication with hardware via **device drivers** |
| **User interface** | Provides CLI or GUI for user interaction |
| **Security** | Manages user logins, permissions and access control |
| **Error handling** | Detects errors and handles them gracefully |

### Types of User Interface

| Type | Description | Advantages | Disadvantages |
|------|-------------|-----------|--------------|
| **GUI** (Graphical User Interface) | Windows, icons, menus, pointer (WIMP) | Easy to use, intuitive | Uses more memory/processing |
| **CLI** (Command Line Interface) | Text commands typed in | Fast, precise, less resource-intensive | Steep learning curve |
| **Menu-driven** | User selects from displayed options | Simple for limited options | Slow for expert users |
| **Voice interface** | Voice commands | Hands-free, accessible | Accuracy issues |

### Device Drivers

> A **device driver** is software that allows the OS to communicate with a specific hardware device.

- Each hardware device (printer, USB device, graphics card) needs its own driver
- Acts as a translator between OS and hardware
- When a new USB device is connected, driver is downloaded automatically (first use)

## 4.3 Interrupts

> An **interrupt** is a signal sent to the CPU to pause the current task and handle an urgent event.

### How Interrupts Work
1. Device or program sends an interrupt signal to the CPU
2. CPU finishes its current instruction
3. **ISR (Interrupt Service Routine)** saves the current state (registers etc.) to a stack
4. CPU processes the interrupt
5. CPU restores saved state and resumes previous task

### Types of Interrupts

| Type | Example |
|------|---------|
| Hardware interrupt | Key pressed on keyboard, mouse click, printer finished |
| Software interrupt | Division by zero, program error |
| Timer interrupt | OS scheduler switching between processes |

### Priority
- Multiple interrupts: handled in **priority order**
- High-priority interrupt can interrupt a lower-priority ISR

## 4.4 Types of Programming Language

| Level | Language type | Examples | Characteristics |
|-------|--------------|---------|----------------|
| **Low level** | Machine code | Binary instructions | Directly executed by CPU; no translation needed; very fast |
| **Low level** | Assembly language | MOV, ADD, SUB (mnemonics) | One-to-one mapping to machine code; CPU-specific |
| **High level** | Procedural | Python, Java, VB.NET, C++ | Human-readable; portable; needs translation |

### Low Level vs High Level

| | Low Level | High Level |
|--|-----------|------------|
| **Readable** | Very hard | Easy |
| **Portable** | No (CPU-specific) | Yes (run on many platforms) |
| **Speed** | Very fast | Slightly slower (due to translation) |
| **Abstraction** | Low (close to hardware) | High (hides hardware details) |
| **Use case** | OS kernels, embedded systems, drivers | Most applications |

## 4.5 Translators

> A **translator** converts source code from one language to machine code.

### Compiler
- Translates the **entire program** at once before execution
- Creates a separate **executable file**
- Errors reported all at once (after full compilation)
- Resulting program runs **fast** (no translation overhead during runtime)
- User cannot see the source code

### Interpreter
- Translates and executes the program **line by line**
- No separate executable created
- Stops at the **first error encountered** (easier debugging)
- Slower execution (translates each time the program runs)
- Source code needed every time program runs

### Assembler
- Translates **assembly language** to machine code
- One-to-one translation (one assembly instruction = one machine code instruction)

| | Compiler | Interpreter | Assembler |
|--|---------|------------|----------|
| Input | High-level source | High-level source | Assembly language |
| Output | Executable file | No file; executes directly | Machine code |
| Speed | Fast execution | Slow execution | Very fast execution |
| Error reporting | All at end | Stops at first error | After assembly |
| Debugging | Harder | Easier | Moderate |

## 4.6 Integrated Development Environment (IDE)

> An **IDE** is a software application that provides a complete set of tools for writing, testing and debugging programs.

### IDE Features

| Feature | Description |
|---------|-------------|
| **Code editor** | Text editor with syntax highlighting, auto-completion |
| **Syntax highlighting** | Colours keywords, strings, comments for readability |
| **Auto-completion** | Suggests variable names, functions as you type |
| **Debugger** | Steps through code, inspects variable values |
| **Breakpoints** | Pause execution at specific lines to inspect state |
| **Run/build tools** | Compile and run code directly from the IDE |
| **Error detection** | Underlines syntax errors before running |
| **Version control integration** | Track changes, collaborate via Git etc. |

### Examples of IDEs
- **Python:** IDLE, PyCharm, VS Code, Thonny
- **Java:** Eclipse, IntelliJ IDEA, NetBeans
- **VB.NET:** Visual Studio
