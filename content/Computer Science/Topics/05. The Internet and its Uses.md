## 5.1 The Internet and the World Wide Web

> The **Internet** is a global network of interconnected computers and networks.

> The **World Wide Web (WWW)** is a collection of web pages and resources stored on web servers, accessed via the internet using HTTP/HTTPS.

| | Internet | World Wide Web |
|--|---------|----------------|
| **What is it?** | The physical/logical network infrastructure | A service (web pages) that runs on top of the internet |
| **Invented by** | ARPANET (1969), TCP/IP (1983) | Tim Berners-Lee (1989–1991) |
| **Protocol** | TCP/IP | HTTP / HTTPS |

### URLs

A **URL** (Uniform Resource Locator) is the address of a web resource.

**Structure:** `https://www.example.com/page.html`

| Part | Example | Meaning |
|------|---------|---------|
| Protocol | `https://` | How data is transmitted |
| Subdomain | `www` | Sub-section of domain |
| Domain name | `example.com` | Human-readable address |
| Path | `/page.html` | Location on the server |

### IP Addresses

> An **IP address** is a numerical label assigned to each device on a network.

- **IPv4:** 32-bit, written as four decimal octets e.g. `192.168.1.1`
- **IPv6:** 128-bit, written in hex e.g. `2001:0db8:85a3::8a2e:0370:7334` — needed because IPv4 addresses are nearly exhausted
- **DNS (Domain Name System):** Translates human-readable domain names (e.g. google.com) to IP addresses

### Key Network Protocols

| Protocol | Full name | Purpose |
|----------|-----------|---------|
| **TCP/IP** | Transmission Control Protocol / Internet Protocol | Suite of protocols for routing packets and ensuring delivery |
| **HTTP** | HyperText Transfer Protocol | Transfer of web pages (unencrypted) |
| **HTTPS** | HTTP Secure | Encrypted web transfer (uses TLS/SSL) |
| **FTP** | File Transfer Protocol | Transferring files between client and server |
| **SMTP** | Simple Mail Transfer Protocol | Sending emails |
| **IMAP/POP3** | Internet Message Access / Post Office Protocol | Receiving emails |

## 5.2 Requesting and Retrieving Web Pages

### How a web page is loaded:

1. User types URL into browser
2. **DNS lookup** — domain name → IP address
3. Browser sends **HTTP GET request** to web server
4. Web server finds the requested resource
5. Server sends response (HTML, CSS, images) back to browser
6. Browser renders the page

### DNS (Domain Name System)

> DNS acts as the internet's phone book — translating domain names to IP addresses.

- **Recursive DNS resolver:** Your ISP's DNS server queries on your behalf
- **Root nameserver:** Top of DNS hierarchy
- **TLD nameserver:** Handles `.com`, `.org`, `.uk` etc.
- **Authoritative nameserver:** Final answer — knows the actual IP for the domain

## 5.3 The Purpose of a Web Browser

> A **web browser** is software that requests, receives and renders web pages.

### Browser Functions
| Function | Description |
|----------|-------------|
| **Sending HTTP requests** | Asks web servers for pages |
| **Rendering HTML/CSS** | Interprets markup and styling to display pages |
| **Executing JavaScript** | Runs client-side scripts for interactivity |
| **Managing cookies** | Stores small text files for session management |
| **Caching** | Stores copies of frequently visited pages/resources locally |
| **Security** | HTTPS certificates, phishing warnings |

### Cookies
- Small text files stored on the user's computer by websites
- Used for: session management (keep you logged in), preferences, tracking
- **First-party cookies:** set by the website you're on
- **Third-party cookies:** set by embedded content (ads/analytics)

## 5.4 Digital Currency

> **Digital (crypto) currency** is a decentralised electronic currency not controlled by any government or bank.

### How it works (Bitcoin example):
- All transactions recorded in a **blockchain** — a distributed, tamper-proof public ledger
- New transactions verified by a network of **miners** (computers solving complex mathematical problems)
- No central authority — operated peer-to-peer

### Advantages
- No central authority — resistant to censorship
- Low transaction fees for international payments
- Transparent blockchain

### Disadvantages
- Highly volatile value
- Used for illegal transactions (anonymous)
- High energy consumption for mining
- Transactions can be slow
- Not widely accepted

## 5.5 Cyber Security

### Cyber Threats

| Threat | Description | Example |
|--------|-------------|---------|
| **Malware** | Malicious software | Virus, worm, trojan, spyware, ransomware |
| **Virus** | Self-replicating code that attaches to files | Infects programs when run |
| **Worm** | Self-replicating; spreads without user action | Spreads via network |
| **Trojan** | Disguised as legitimate software | Backdoor allows remote access |
| **Ransomware** | Encrypts files; demands payment | WannaCry attack |
| **Spyware** | Secretly monitors activity | Keyloggers |
| **Phishing** | Fake emails/sites trick users into giving credentials | Fake bank email |
| **Pharming** | Redirects legitimate URL to fake site | DNS poisoning |
| **SQL injection** | Injecting malicious SQL into input fields | Bypass login, extract data |
| **DDoS** | Distributed Denial of Service — flood server with traffic | Overwhelm website |
| **Brute force** | Try all possible passwords systematically | Password cracking |
| **Social engineering** | Manipulating people to reveal information | Pretexting, baiting |

### Cyber Security Measures

| Measure | How it helps |
|---------|-------------|
| **Anti-malware/antivirus** | Detects and removes malicious software |
| **Firewall** | Filters network traffic; blocks unauthorised access |
| **Strong passwords** | Harder to brute-force |
| **Two-factor authentication (2FA)** | Second verification step; harder to compromise |
| **Encryption** | Protects data in transit and at rest |
| **Access levels/permissions** | Limits damage if one account is compromised |
| **Regular updates/patches** | Closes known vulnerabilities |
| **User education** | Teaches users to recognise phishing etc. |
| **HTTPS** | Encrypts web traffic |
| **Proxy server** | Can filter and monitor outgoing/incoming traffic |

### Network Security Concepts

| Term | Definition |
|------|-----------|
| **Authentication** | Verifying identity (username + password, biometrics) |
| **Authorisation** | Determining what an authenticated user is allowed to do |
| **VPN** | Virtual Private Network — creates encrypted tunnel over internet |
| **SSL/TLS** | Protocols for encrypted web communication (used in HTTPS) |
