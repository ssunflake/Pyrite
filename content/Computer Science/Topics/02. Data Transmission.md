## 2.1 The Structure of a Data Packet

> A **data packet** is a fixed-size chunk of data that is transmitted across a network.

When a large file is sent across a network it is split into many packets. Each packet travels independently and is reassembled at the destination.

### Packet Structure

| Part | Contents | Purpose |
|------|----------|---------|
| **Header** | Sender IP, destination IP, packet number, total packets | Identifies where the packet came from and where it's going |
| **Payload (body)** | Actual data being sent | The content |
| **Trailer** | Error detection data (e.g. checksum) | Detects errors in transmission |

## 2.2 Packet Switching

> **Packet switching** is a method of data transmission where each packet can take a **different route** through the network to reach its destination.

### How it works:
1. Data split into packets
2. Each packet routed independently by **routers**
3. Routers use routing tables to find the best available path
4. Packets may arrive **out of order**
5. Destination device **reassembles** them using the packet numbers

### Advantages of Packet Switching
- Network is fault-tolerant: if one route fails, packets take another
- Efficient use of network — no single path is reserved
- Multiple packets can travel simultaneously

### Routers
- Devices that **route packets** between networks
- Read the destination IP in each packet's header
- Choose the best path based on availability and speed

## 2.3 Methods of Data Transmission

### Serial vs Parallel

| Feature | Serial | Parallel |
|---------|--------|---------|
| Wires | 1 wire | Multiple wires |
| Bits sent at a time | One at a time | Multiple simultaneously |
| Speed | Slower | Faster |
| Data order | Arrives in order | Can arrive out of order (**skew**) |
| Interference | Less | More |
| Cost | Cheaper cable | More expensive |
| Distance | Suitable for long distances | Limited to ~5 metres |
| Example | USB, Ethernet | Internal computer buses |

> **Skew:** In parallel transmission, bits can arrive at different times due to slight differences in wire length/quality, causing data corruption.

### Simplex, Half-Duplex, Full-Duplex

| Type | Direction | Simultaneous? | Example |
|------|-----------|--------------|---------|
| **Simplex** | One direction only | N/A | Keyboard → computer |
| **Half-duplex** | Both directions, but one at a time | No | Walkie-talkie, Wi-Fi |
| **Full-duplex** | Both directions at the same time | Yes | Phone call, Ethernet |

**Wi-Fi uses half-duplex** — a device can send or receive at any moment, but not both at exactly the same time.

## 2.4 The Universal Serial Bus (USB)

> **USB** (Universal Serial Bus) is an industry standard interface for connecting devices to computers.

| Component | Description |
|-----------|------------|
| **USB port** | Socket on device for inserting USB cable |
| **USB cable** | Transmission media using USB standard |
| **USB device** | Device that plugs into a USB port (e.g. memory stick) |
| **USB connection** | USB cable + port together transferring data |

### USB Advantages
- **Universal standard** — compatible with almost all devices
- **Plug and play** — automatically detected, drivers download automatically
- Can power/charge devices (no separate power source needed)
- Relatively fast data transfer

### USB Disadvantages
- Limited cable length (~5 metres)
- Slower than some other interfaces (e.g. Thunderbolt, Ethernet)
- USB connection is serial (one bit at a time) — though modern versions achieve high speeds via compression and multiplexing

## 2.5 Detecting Errors in Data Transmission

> Transmission errors occur when interference causes bits to change, be lost, or be added. Error detection methods check for these mistakes.

### Parity Check

Each byte has 7 data bits + 1 **parity bit** (extra bit to make total 1s odd or even).

- **Even parity:** total number of 1s in the byte (including parity bit) is **even**
- **Odd parity:** total number of 1s is **odd**

**Example (even parity):**
- Data bits: `0100101` → three 1s (odd)
- Parity bit needed: **1** (to make four 1s = even)
- Full byte: `10100101`

**Limitation:** Can only detect an **odd** number of bit errors — if two bits flip, the parity still looks correct.

### Checksum

- A value calculated from the data before transmission
- Transmitted alongside the data
- Receiving device recalculates — if values match, no error; if different, error detected
- More powerful than parity — detects multiple-bit errors

### Echo Check

- Receiving device **sends the data back** to the sender
- Sender compares received echo to original
- If they match, no error; if different, error detected
- Doubles bandwidth use but very reliable

| Method | How | Detects |
|--------|-----|---------|
| Parity check | Count 1s, add parity bit | Single bit errors |
| Checksum | Mathematical calculation on data | Multiple bit errors |
| Echo check | Data sent back to sender | Any difference |

## 2.6 Encryption

> **Encryption** is the process of scrambling data so it cannot be understood by anyone who intercepts it without the decryption key.

### Why Encrypt?
- Protects data during transmission over networks
- Even if intercepted, encrypted data is unreadable
- Used in: HTTPS, messaging apps, banking, email

### Symmetric Encryption
- **Same key** used to encrypt and decrypt
- Fast, but key must be securely shared between parties
- Risk: if key is intercepted, all data is compromised
- Example: AES (Advanced Encryption Standard)

### Asymmetric Encryption (Public Key Encryption)
- Two keys: **public key** (anyone can use to encrypt) and **private key** (only owner can decrypt)
- No need to securely share a key
- Slower than symmetric
- Used in: HTTPS, digital signatures
- Example: RSA

### Key Terms
| Term | Definition |
|------|-----------|
| **Plaintext** | Original, unencrypted data |
| **Ciphertext** | Encrypted data |
| **Encryption key** | Data used to encrypt the plaintext |
| **Decryption** | Reversing encryption to recover plaintext |

## Key Definitions

| Term | Definition |
|------|-----------|
| **Data packet** | Unit of data transmitted across a network, containing header, payload, and trailer |
| **Packet switching** | Routing packets independently through a network; may take different paths |
| **Serial transmission** | Data sent one bit at a time along a single wire |
| **Parallel transmission** | Multiple bits sent simultaneously along multiple wires |
| **Simplex** | Data transmission in one direction only |
| **Half-duplex** | Two-way transmission, but only one direction at a time |
| **Full-duplex** | Simultaneous two-way data transmission |
| **USB** | Universal Serial Bus — industry standard for connecting devices |
| **Parity check** | Error detection using an extra bit to keep total 1s odd or even |
| **Checksum** | Error detection using a calculated value compared before and after transmission |
| **Encryption** | Converting data to an unreadable form to protect it during transmission |
